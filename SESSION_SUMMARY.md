## 📊 WHAT IT DOES

### User Flow:

1. **User visits game page** (e.g., `/games/A Link to the Past`)
2. **Clicks "Create YAML"** button
3. **System loads game options** from `AutoWorldRegister.world_types`
4. **Renders dynamic form** with all game-specific options
5. **User fills form** (intuitive UI with tooltips)
6. **User submits** with one of two actions:
   - **Download YAML**: Gets .yaml file
   - **Save to Vault**: Stores in database for lobby use

### Example: A Link to the Past

```
Form displays automatically:
├── Player Information
│   ├── Player Name: [text input]
│   └── Description: [text input]
├── Game Options
│   ├── Difficulty: [Easy | Normal | Hard] (dropdown)
│   ├── Goal: [Ganon | Pedestal | All Dungeons] (dropdown)
│   ├── Crystals Required: [0-7] (slider)
│   ├── Enable Hard Mode: [No | Yes] (toggle buttons)
│   └── ... (50+ more options)
└── [Cancel] [Save to Vault] [Download YAML]
```

## 🔧 TECHNICAL ARCHITECTURE

### How It Works:

```
Frontend (yaml_creator.html)
    ↓ fetch('/api/games/ALttP/options')

Backend (main.py)
    ↓ get_game_options('ALttP')

yaml_creator.py
    ↓ AutoWorldRegister.world_types['ALttP']
    ↓ Options.get_option_groups(world)
    ↓ build_option_metadata(option_class)

Returns JSON:
{
  "game_name": "A Link to the Past",
  "option_groups": {
    "Game Options": {
      "difficulty": {
        "type": "choice",
        "choices": ["easy", "normal", "hard"],
        "default": "normal",
        "description": "Game difficulty level"
      }
    }
  }
}
    ↓

Frontend renders form dynamically
User submits
    ↓ POST('/api/games/ALttP/create-yaml', formData)

Backend (main.py)
    ↓ form_data_to_yaml(formData)

yaml_creator.py
    ↓ Converts to YAML format
    ↓ validate_and_save_yaml() (if saving)

Returns:
- Download: YAML file
- Save: {success: true, yaml_id: 123}
```

## 🎓 KEY INSIGHTS

### 1. Archipelago's Options System is Self-Describing

Every Archipelago game defines its options like this:

```python
class ALttPWorld(World):
    options_dataclass = ALttPOptions

@dataclass
class ALttPOptions(PerGameCommonOptions):
    difficulty: Difficulty
    goal: Goal
    # ... more options

class Difficulty(Choice):
    """Game difficulty level"""  # ← Form description
    option_easy = 0              # ← Form options
    option_normal = 1
    option_hard = 2
    default = 1                  # ← Pre-selected
```

We just read this and generate forms automatically!

### 2. Dynamic > Static

**Alternative approach** (bad):
- Create 100+ static HTML forms manually
- Update when games change
- Maintain forever

**Our approach** (good):
- ONE form generator
- Reads game definitions
- Automatically supports new games
- Zero maintenance

### 3. WebHostLib Already Solved This

We adapted WebHostLib's pattern:
- They use server-side Jinja templates
- We use JSON API + JavaScript
- Both approaches work
- Ours is more flexible for modern web dev
