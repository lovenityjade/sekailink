# Sekaiemu Core Settings Reference

Date: 2026-06-27  
Scope: Sekaiemu/libretro cores currently present or referenced by SekaiLink BETA-3 runtime modules.  
Purpose: provide a clean source for the Client Core `Core Settings` UI, close to what RetroArch exposes, without making the user read raw libretro option keys.

## Important Notes

- Sekaiemu already has a runtime core option manager:
  - `apps/sekaiemu/src/core_option_manager.cpp`
  - `apps/sekaiemu/src/core_option_manager.hpp`
  - `apps/sekaiemu/src/runtime_menu_core_settings.cpp`
- Sekaiemu persists core options under:
  - `save_dir/core-config/<core_name>.cfg`
  - `save_dir/core-config/<core_name>.options.txt`
  - optional game override: `save_dir/core-config/<core_name>__<game_name>.cfg`
- The current Client Core settings page contains a useful draft table, but some keys are UI-side planned keys and do not exactly match the libretro runtime keys.
- For production, the ideal source of truth is the `.options.txt` generated by Sekaiemu after loading the core, because it comes from the core's own libretro metadata.
- Options marked as restart-sensitive should be saved immediately but may require restarting the loaded core/game to fully apply.

## UI Policy

Recommended Client Core layout:

- **Simple mode** should show only settings a normal player would understand and that are unlikely to break compatibility.
- **Advanced mode** should show accuracy, renderer, overclocking, BIOS, palette, and experimental options.
- Settings must be wired. Do not add decorative unimplemented controls.
- Core-specific settings should use the true libretro key when saved, not a friendly alias, unless the alias is translated before persistence.
- Every option row should expose:
  - Display label
  - Help text
  - Current value
  - Default value
  - Restart required indicator, when known

## SNES: bsnes-mercury-performance

Libretro info:

- Display name: `Nintendo - SNES / SFC (bsnes-mercury Performance)`
- Supported extensions: `sfc`, `smc`, `bml`, `gb`, `gbc`, `st`, `bs`
- License: GPLv3
- Notes: performance profile is faster but has known Cx4 issues, so Mega Man X2/X3 should prefer `snes9x`.

Runtime schema already captured by Sekaiemu from a live run:

| Key | Label | Default | Values | Mode | Restart |
|---|---|---:|---|---|---|
| `bsnes_region` | System region | `auto` | `auto`, `ntsc`, `pal` | Simple | Usually yes |
| `bsnes_aspect_ratio` | Aspect ratio | `auto` | `auto`, `ntsc`, `pal` | Simple | No |
| `bsnes_crop_overscan` | Crop overscan | `disabled` | `disabled`, `enabled` | Simple | No |
| `bsnes_superfx_overclock` | SuperFX speed | `100%` | `100%`, `150%`, `200%`, `300%`, `400%`, `500%`, `1000%` | Advanced | Maybe |
| `bsnes_chip_hle` | Special chip accuracy | `LLE` | `LLE`, `HLE` | Advanced | Maybe |
| `bsnes_violate_accuracy` | Allow reduced accuracy | `disabled` | `disabled`, `enabled` | Advanced | Maybe |
| `bsnes_gamma_ramp` | Gamma ramp | `disabled` | `disabled`, `enabled` | Advanced | Likely |

Recommended SekaiLink defaults:

```ini
bsnes_region=auto
bsnes_aspect_ratio=auto
bsnes_crop_overscan=disabled
bsnes_superfx_overclock=100%
bsnes_chip_hle=LLE
bsnes_violate_accuracy=disabled
bsnes_gamma_ramp=disabled
```

Compatibility notes:

- Good default for ALttP, SM, SMZ3, EarthBound, DKC, KDL3, SMW, Lufia II.
- Do not use as default for Mega Man X2/X3 because the installed info file warns about Cx4 compatibility.

## Settings Page Implementation Checklist

1. Use the real libretro option key for persistence.
2. Display friendly labels in the UI, but save the key exactly as the core expects.
3. Load generated schema from `save_dir/core-config/<core>.options.txt` whenever available.
4. Fall back to this document's recommended subset if the schema has not been generated yet.
5. Keep game-specific overrides separate from core defaults:
   - core-wide: `<core_name>.cfg`
   - game-specific: `<core_name>__<game_name>.cfg`
6. Never expose debug layer/channel toggles in Simple mode.
7. For BETA-3 release, prefer stable defaults over exhaustive RetroArch parity.

## Known Gaps To Resolve

- Client Core currently has a draft `coreSettingsProfiles` table, but it is not guaranteed to match the exact generated runtime schema for every core.
- `nestopia` and `snes9x` should be added explicitly to Client Core settings if they remain active runtime cores.
- `fceumm` exists in the bundle but current NES manifests use `nestopia`; decide whether both should be configurable or whether FCEUmm should be hidden as a fallback.
- N64 settings are present as a future profile, but N64 is not BETA-3 enabled.
- Dolphin/SoH/SM64EX settings belong in separate runtime-specific docs, not in this libretro core settings document.
