# Live Link Deployment Notes

Date: 2026-06-11

Server: `link-vps`

## Purpose

Restore the legacy social-bots Discord bug-report backend and connect the public
Link API bug-report endpoint to it.

## Changes Applied

- Repaired `sekailink-social-bots.service`.
  - Previous unit pointed to missing `/opt/sekailink-social-bots`.
  - Active unit now runs from `/opt/sekailink/services/social-bots`.
  - Python virtual environment created at `/opt/sekailink/services/social-bots/.venv`.
  - Service is enabled and running on `0.0.0.0:8095`.
- Added compatibility symlink:
  - `/opt/sekailink-social-bots -> /opt/sekailink/services/social-bots`
  - This preserves hardcoded state paths in the existing Discord bot module.
- Disabled the social-bots AI writer in the live `.env`:
  - `SEKAILINK_AI_ENABLED=false`
  - Reason: current `fastembed` no longer supports the configured
    `intfloat/multilingual-e5-small` model. Bug reports do not require AI memory.
- Patched live `ai_writer.py` so disabled AI no longer initializes semantic
  memory or `fastembed`.
- Rebuilt and installed `sekailink-chat-api.service` from current
  `services/link-social` sources.
- Added the social-bots forwarding config to
  `/opt/sekailink/link/chat-api/config/chat_api.json`.
