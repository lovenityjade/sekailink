# Debug Session - 2026-07-02 06:57 EDT

Context: Super Metroid launch started Sekaiemu, Nintendo logo began to render, then Sekaiemu closed immediately. Root cause was a combination of incomplete SHA256 ROM checksum support in Client Core and Linux `SIGPIPE` exposure in Sekaiemu runtime memory socket writes.

## Bugs corrected

- Added robust ROM checksum normalization for MD5/SHA1/SHA256/SHA3-256 in Client Core runtime module import/validation.
- Added SHA256 support to ROM requirement matching and auto-detection.
- Added Super Metroid clean base ROM SHA256 to the Super Metroid and SMZ3 manifests. Affected games: Super Metroid, SMZ3.
- Hardened Archipelago client shutdown writes so a closed stdin does not surface as an uncaught `EPIPE` during coupled runtime teardown.
- Fixed `runtime-module-library.cjs` default `processRef` initialization so the module can be instantiated without passing `processRef`.
- Hardened Sekaiemu runtime memory socket writes on Linux by ignoring `SIGPIPE` and using `MSG_NOSIGNAL` where available.
- Rebuilt `sekaiemu_libretro_spike` and copied it into `runtime/platforms/linux-x64/bin/`.
- Fixed Runtime Activity item text parsing so unquoted multi-word fields like `item_name=Power Bombs` keep the full item name instead of stopping at the first word.
- Fixed Runtime Activity rendering so long titles/details wrap cleanly inside the compact window.
- Fixed Runtime Activity history retention so the window can show the sync history across runtime social window reopen/rejoin flows instead of only the newest short slice.
- Fixed ROM Library checksum status so empty checksum arrays no longer appear as verified checksum support.
- Fixed Secret of Evermore ROM import state: the visual catalog no longer marks it as `In learning`, and the runtime module now includes the official APWorld MD5 for `Secret of Evermore (USA).sfc`. Affected games: Secret of Evermore.
- Fixed Secret of Evermore generation on the Worlds generator by installing the validated base ROM under both `/opt/sekailink-generate/roms/soe_usa.sfc` and the active service tree `/opt/sekailink/worlds/generation-server/roms/soe_usa.sfc`, then changing both live `soe_options.rom_file` values from relative filenames to absolute generator-local paths. Restarted `sekailink-generation-server.service`. Updated canonical `runtime/ap/host.yaml` to point at the imported local ROM path. Affected games: Secret of Evermore.
- Fixed Secret of Evermore send/reception startup by normalizing web AP client room addresses to explicit `ws://host:port` and disabling the encoded `#server=` autoconnect hash by default. The upstream SoE web client does not decode the hash value and otherwise tries invalid/SSL-only addresses like `wss://link.sekailink.com%3A38358:38281`. Affected games: Secret of Evermore and other `web_ap_client` runtime modules.
- Fixed a Sekaiemu OpenGL software-frame resolution bug affecting SNES hires/menu screens: dynamic widened frames now use a separate active internal frame size while keeping the Sekaiemu window stable, matching RetroArch-style viewport behavior more closely. Affected games: Secret of Mana, especially name selection/save selection/saving screens.
- Fixed a Sekaiemu gamepad/UI focus conflict where Xbox controller buttons, including X, could drive ImGui navigation and focus/open HUD widgets instead of staying gameplay-only. ImGui no longer enables gamepad navigation and controller/joystick events are filtered before ImGui processing. Affected surfaces: Sekaiemu controller input/HUD.
- Fixed the ROM import carousel filter/state so supported catalog games appear without requiring `All games`, `A Link to the Past` resolves through the `alttp` module alias, and Ready/Verified cards are not globally dimmed by module lookup state. Affected games: all ROM-imported games.
- Fixed launch double-submit windows by opening launch feedback immediately, locking the launch button before runtime prep resolves, and guarding launch-selection/tracker-variant modal actions against repeated clicks.
- Added an ALttP receive-slot watchdog for the reported menu/start desync: if the ROM receive slot stays occupied for 15 seconds in stable gameplay while additional received items are queued, the client logs the stuck state and clears the stale slot so the queue can continue on the next tick. Also fixed the ALttP endgame mode comparison to use the byte value instead of the bytes object. Affected games: A Link to the Past.
- Normalized short ALttP item labels in Runtime Activity/toasters so `Book`, `Fire`, and `Ice` render as `Book of Mudora`, `Fire Rod`, and `Ice Rod`; generic key labels such as `Small` now prefer specific mapped labels like `Small Key (Eastern Palace)` when keysanity metadata is present and only fall back to `Small Key` when no dungeon context exists. Affected games: A Link to the Past and any trace source using these short labels.
- Fixed Game Boy Color launches through Sekaiemu/Gambatte by writing a launch-local core option override `gambatte_gb_hwmode=GBC` for modules whose `sekaiemu.core_id` is `gbc`. This prevents GBC-only games from falling back into DMG/GB mode and showing hardware warning screens after boot. Affected games: Pokemon Crystal, Link's Awakening DX, Oracle of Ages, Oracle of Seasons, and other Sekaiemu modules using Gambatte as `gbc`.
- Fixed Pokemon GBA runtime connection gating: Pokemon Emerald and Pokemon FireRed/LeafGreen were launching Sekaiemu but Client Core refused to start the Archipelago runtime client because `runtime/game-registry/archipelago-clients.json` still marked both entries `runtime_unavailable`. They are now enabled as Canari test targets in the AP client registry, game registry, and runtime manifests. Affected games: Pokemon Emerald, Pokemon FireRed and LeafGreen.
- Reduced severe mGBA log spam in Sekaiemu by suppressing libretro `debug` log lines and repeated identical `SRAM memory size:` info lines after the first occurrence. This keeps true warnings/errors visible while preventing Pokemon GBA launches from flooding Client Core with DMA/SRAM noise. Affected games: Pokemon Emerald, Pokemon FireRed and LeafGreen, and other mGBA/GBA modules.
- Fixed Pokemon GB/GBC runtime connection gating: Pokemon Crystal and Pokemon Red/Blue were launching Sekaiemu/Gambatte but Client Core refused to start their Archipelago runtime clients because the AP client registry still marked both entries `runtime_unavailable`. They are now enabled as Canari test targets in the AP client registry, game registry, and runtime manifests. Affected games: Pokemon Crystal, Pokemon Red and Blue.
- Hardened Sekaiemu/Gambatte runtime memory access for GB/GBC AP clients by adding explicit Game Boy ROM, WRAM, CartRAM, and 64 KiB System Bus resolution instead of relying only on libretro memory maps. This should make BizHawk-compatible reads/writes deterministic for Pokemon GB/GBC clients and reduce connection/write failures. Affected games: Pokemon Crystal, Pokemon Red and Blue, and future Gambatte GB/GBC runtime clients.
- Removed browser-style controls from Client Core Electron windows by disabling DevTools and blocking back/forward/reload/developer-console accelerators (`Alt+Left/Right`, browser back/forward keys, `F5`, `Ctrl/Cmd+R`, `F12`, and `Ctrl/Cmd+Shift+I/J/C`) on the main app, dashboard, runtime social, Web AP, and web tracker windows. Affected surfaces: Client Core shell and external Client Core windows.
- Enabled Oracle of Ages as a Canari GBC isolation test while leaving Oracle of Seasons unavailable. Purpose: compare against Pokemon Crystal's remaining `Game Boy Color only` launch failure to determine whether the issue is generalized to Sekaiemu/Gambatte GBC modules or specific to Pokemon Crystal. Affected games: The Legend of Zelda - Oracle of Ages.
