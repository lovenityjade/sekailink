# Debug Session - 2026-07-11 00:48 EDT

Context: Weekly stabilization pass covering community bug reports and usability requests before the next SekaiLink update.

## Bugs corrected

- Restored authenticated Client Core sessions after security maintenance. Link's Chat API still targeted Nexus's former public identity listener, which is now correctly closed by the firewall; it now validates sessions through the dedicated private SSH tunnel at `127.0.0.1:19295`. The canonical defaults, deployment example and service documentation were updated to prevent recurrence. Affected surface: Client Core session restoration, chat, lobbies and all Link routes that validate Nexus sessions.
- Fixed the public Worlds TLS endpoint: `worlds.sekailink.com` previously fell through to the APTest virtual host and served a certificate valid only for `aptest.sekailink.com`. Worlds now has its own Nginx virtual host and automatically renewed Let's Encrypt certificate while preserving the existing generation backend.
- Capitalized public profile role labels in Client Core so the default account role renders as `Player` instead of `player`.
- Fixed a direct-message refresh loop discovered during Client Core restart: clearing a conversation's unread map retriggered the DM effect immediately, causing continuous message loads/read acknowledgements and heavy log/network spam. DM polling now stays on its intended 10-second interval and clears the initial unread count only once per opened conversation. Affected surface: Client Core friend chat panel.
- Replaced the friend-list right-click placeholder toast with a real anchored context menu. The menu now exposes View Profile, Message, Open Public Profile, Remove Friend, and Block; destructive actions require confirmation and call the existing Nexus social endpoints. Affected surface: Client Core friend list.
- Fixed multi-config generation handoff matching so generated launch artifacts are assigned by exact Archipelago slot name or numeric slot before any fuzzy player-name fallback. Previously, multiple games owned by the same user could all attach to the first config, leaving later games with `Downloads: 0` and `No generated Archipelago slot was found`. Deployed the server-side handoff fix to Link; existing generations with already-persisted bad indexes still require regeneration. Affected games: all solo and multiplayer lobbies where one account owns multiple seed configs; originally reported with The Minish Cap and A Link to the Past.
