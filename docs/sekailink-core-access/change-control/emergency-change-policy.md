# Emergency Change Policy

Emergency live changes are allowed only when production, stream, login, release
or live room operation is actively blocked.

## Procedure

1. Stabilize with the smallest possible action.
2. Capture command, target and reason.
3. Patch if needed.
4. Document immediately in the same session.
5. Create or update change record before final response.
6. Add rollback and verification steps.
