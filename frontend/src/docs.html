API Documentation

The SekaiLink API is a REST API for building bots, trackers, and integrations. All endpoints return JSON and follow RESTful conventions.

Endpoints

  • Authentication
  • Games
  • Lobbies
  • Users
  • YAML Manager
  • WebSocket Events
  • Error Handling

Base URL

https://sekailink.xyz/api

All API requests should be prefixed with this base URL.

Authentication

SekaiLink uses session-based authentication via Discord OAuth2. Most endpoints require authentication.

OAuth Flow

GET /auth/discord
Initiates Discord OAuth2 flow. Redirects to Discord for authorization.
GET /auth/callback
OAuth callback endpoint. Creates session cookie.
GET /auth/logout
Destroys session and logs out user.

Session Cookies

Include credentials: 'include' in fetch requests to send session cookies: 

fetch('/api/lobbies', {
    credentials: 'include'
})

Games

GET /api/games

List all available games with filters.

Query Parameters:
  • search - Search by game name
  • requires_rom - Filter by ROM requirement (true/false)
  • limit - Limit results (default: 50)
  • offset - Pagination offset
Example Response: 
[
    {
        "slug": "a-link-to-the-past",
        "name": "The Legend of Zelda: A Link to the Past",
        "description": "Classic SNES adventure",
        "requires_rom": true,
        "boxart_url": "/static/boxarts/alttp.png",
        "sync_count": 1234
    }
]
GET /api/games/:slug

Get detailed information about a specific game.

POST /api/games/:slug/favorite

Toggle favorite status for a game. Requires authentication.

GET /api/games/:slug/lobbies

Get lobbies for a specific game.

Query Parameters:
  • status - Filter by status (active, finished, cancelled)

Lobbies

GET /api/lobbies

List all active lobbies.

Example Response: 
[
    {
        "id": 123,
        "slug": "epic-game-1234",
        "name": "Epic Multiworld Adventure",
        "host_name": "JohnDoe",
        "max_players": 10,
        "player_count": 5,
        "state": "open",
        "created_at": "2026-01-03T12:00:00Z"
    }
]
POST /api/lobbies/create

Create a new lobby. Requires authentication.

Request Body: 
{
    "name": "My Lobby",
    "max_players": 10,
    "visibility": "open",
    "rules": "Be respectful!",
    "time_limit_hours": 3,
    "restrict_time_limit": true,
    "settings": {
        "allow_multigame": true,
        "disallow_rom_games": false,
        "allow_broadcast": true,
        "blacklisted_games": ["game-slug"]
    }
}
GET /api/lobbies/:id

Get detailed lobby information including players.

POST /api/lobbies/:id/join

Join a lobby. Requires authentication.

Request Body: 
{
    "yaml_id": 456
}
POST /api/lobbies/:id/ready

Toggle ready status. Requires authentication.

DELETE /api/lobbies/:id

Delete a lobby. Host only.

YAML Manager

GET /api/yaml

List user's YAMLs. Requires authentication.

POST /api/yaml/upload

Upload a YAML file. Requires authentication.

DELETE /api/yaml/:id

Delete a YAML. Requires authentication.

WebSocket Events

Real-time updates use Socket.IO. Connect to wss://sekailink.xyz with Socket.IO client.

Event Format

{
    "type": "event_name",
    "data": { ... },
    "version": 1
}

Lobby Events

  • lobby_updated - Lobby settings or state changed
  • player_joined - New player joined
  • player_left - Player left
  • player_ready - Player readied up
  • generation_started - Seed generation began
  • generation_progress - Generation progress update
  • generation_complete - Generation finished
  • timer_started - Sync timer started
  • chat_message - New chat message

Example Usage

const socket = io('https://sekailink.xyz');

socket.on('connect', () => {
    socket.emit('join_lobby', { lobby_id: 123 });
});

socket.on('lobby_updated', (data) => {
    console.log('Lobby updated:', data);
});

Error Handling

HTTP Status Codes

Code Meaning
200 Success
201 Created
400 Bad Request - Invalid parameters
401 Unauthorized - Not logged in
403 Forbidden - Insufficient permissions
404 Not Found
429 Too Many Requests - Rate limited
500 Internal Server Error

Error Response Format

{
    "error": "Error message here",
    "code": "ERROR_CODE",
    "details": { ... }
}

Rate Limiting

API requests are rate limited to prevent abuse:

  • Authenticated users: 100 requests per minute
  • Unauthenticated: 20 requests per minute
  • WebSocket: 50 events per minute

Rate limit headers are included in all API responses: 

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1641024000
